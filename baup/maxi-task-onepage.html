MAXI TASK

เคสสตัดดี้การออกแบบระบบโครงสร้างพื้นฐานโทรคมนาคม ที่ต้องเชื่อม UX, engineering และข้อกำกับทางกฎหมาย ให้ระบบซับซ้อนกลายเป็น workflow ที่ออกแบบได้ สร้างได้ และยังไม่หลุด compliance

Year 2023
Role UX/UI bridge with engineering
Tool Adobe XD
Category UI/UX, telecom infrastructure
Problem

ปัญหาไม่ใช่ข้อมูลเยอะ แต่ข้อมูลสำคัญมองไม่เห็นตอนที่ต้องตัดสินใจ

ระบบโครงข่ายโทรคมนาคมมีความซับซ้อนโดยธรรมชาติ แต่ pain point จริงคือระบบบังคับให้ผู้ใช้ต้องจำ แทนที่จะเห็น ข้อมูลกระจายอยู่หลายหน้าจอ ทำให้การ trace ปัญหา network ใช้เวลานาน

Field team, operations และ management จึงตีความระบบเดียวกันคนละแบบ การตัดสินใจสำคัญถูกเลื่อนออกไป เพราะทีมต้องอ่านและแปลข้อมูล แทนที่จะเข้าใจภาพรวมได้ทันที

Hidden context

ข้อมูลสำคัญอยู่คนละหน้าจอ ทำให้ผู้ใช้ต้องประกอบบริบทด้วยตัวเอง

Slow tracing

การไล่ network issue ใช้เวลานานเพราะ flow ไม่ต่อเนื่องจาก map ไป asset และ core network

Different mental models

แต่ละทีมเห็นและตีความระบบไม่เหมือนกัน ทำให้การสื่อสารข้ามทีมช้าลง

Compliance pressure

การออกแบบต้องรองรับข้อกำกับและกฎหมายไทยโดยไม่ทำให้ workflow ใช้งานยากขึ้น

Solution

จัดโครงสร้างข้อมูลใหม่ให้ระบบโชว์สิ่งที่สำคัญในจังหวะที่ต้องใช้

แนวทางหลักคือยุบข้อมูลที่กระจัดกระจายให้กลายเป็น context-rich single views และออกแบบ UI เพื่อการ scan ด้วยสายตา แทนการอ่านข้อความหนัก ๆ ในทุกขั้นตอน

ผมเชื่อม Map, Asset และ Core network ให้เป็น mental model เดียวกัน เพื่อเปลี่ยนงาน tracing จากการกระโดดข้ามหน้าจอให้กลายเป็น flow ต่อเนื่อง

Context-rich views Map + asset model Network tracing Visual scanning Regulatory constraints
Workflow

จาก raw data สู่ service insight ที่ทีมใช้ตัดสินใจได้เร็วขึ้น

Surface the right data

แยกข้อมูลที่ต้องเห็นทันทีออกจากข้อมูลที่เป็นรายละเอียด เพื่อให้ผู้ใช้ไม่ต้องจำทุกอย่างเอง

Unify map and asset

รวมมุมมอง GIS, network asset และ core network ให้ผู้ใช้ trace งานใน flow เดียว

Design for scanning

จัดลำดับข้อมูล สี สถานะ และ interaction ให้เหมาะกับงานตรวจสอบ ไม่ใช่แค่หน้าจออ่านข้อมูล

Outcome

ระบบที่ซับซ้อนเริ่มอ่านได้ เพราะทีมเห็นโครงสร้างเดียวกัน

  • ลดการกระโดดข้ามหลายหน้าจอ ด้วยมุมมองที่รวม context สำคัญไว้ในจุดเดียว
  • ช่วยให้ field team, operation และ management ใช้ภาพรวมเดียวกันในการไล่ปัญหา
  • เปลี่ยนงาน tracing ให้ต่อเนื่องขึ้นจาก Map ไป Asset และ Core network
  • ทำให้ข้อมูลที่เคยต้องอ่านเยอะ ถูกจัดให้ scan ได้ง่ายและเห็นความสัมพันธ์เร็วขึ้น
  • คุมข้อจำกัดด้าน engineering และกฎหมายไทยไปพร้อมกับการทำให้ระบบ buildable

ในระบบ infrastructure ที่ซับซ้อน UX ที่ดีคือการทำให้ทีม เห็นความสัมพันธ์ก่อนต้องตีความเอง
