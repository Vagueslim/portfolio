WCF Digital

เคสสตัดดี้การออกแบบระบบหลักสำหรับจัดการเงินทดแทนจากการบาดเจ็บหรือเจ็บป่วยจากการทำงาน ครอบคลุมตั้งแต่รับเรื่อง ตรวจเอกสาร วินิจฉัย ไปจนถึงการจ่ายเงินชดเชย

Year 2025
Role UX/UI design, system foundation
Tool Figma
Category Government service design
Problem

ระบบเดิมอายุยาวนาน แต่การทำงานจริงกระจายอยู่หลายบทบาท

WCF เป็นระบบแกนกลางของสำนักงานประกันสังคมสำหรับงานเงินทดแทนที่มีขั้นตอนซับซ้อนและมีข้อกำหนดเฉพาะทางกฎหมายจำนวนมาก งานที่พบไม่ได้เป็นเพียงหน้าจอไม่ครบ แต่เป็นปัญหาโครงสร้างที่แต่ละทีมเดินหน้าแยกกัน

โจทย์สำคัญคือทำให้กระบวนการจากเอกสารจริงเข้าสู่ระบบได้ชัดขึ้น ลดการพึ่งพาความจำของเจ้าหน้าที่ และทำให้สถานะงานกับการส่งต่อระหว่างหน่วยไม่หลุดหาย

Legacy logic

กฎงานสะสมมานาน ทำให้การออกแบบต้องคุมทั้งข้อบังคับและวิธีทำงานจริง

Fragmented flow

หลายบทบาทมีทางเดินงานของตัวเอง ทำให้ขอบเขตการรับผิดชอบไม่ชัด

Manual workaround

เจ้าหน้าที่ต้องจำขั้นตอนหรือทำงานนอกระบบเพื่อให้เคสเดินต่อได้

Slow progression

ความคลุมเครือของสถานะทำให้เคสคืบหน้าช้าและมีโอกาสผิดพลาดสูง

Solution

เปลี่ยนงานที่แตกเป็นส่วน ๆ ให้กลายเป็นระบบที่ทุกทีมสร้างต่อได้

แนวทางหลักคือกำหนด workflow แบบ case-centered ให้แต่ละเคสมีสถานะชัดเจน พร้อมโครงสร้าง ownership และ handoff ที่ตรวจสอบย้อนหลังได้

งานออกแบบถูกยกระดับเป็น foundation กลาง ทั้ง pattern, variant, component logic และ Storybook เพื่อให้ทีม vendor และทีมพัฒนาทำงานจากแหล่งอ้างอิงเดียวกัน

Case-centered workflow Shared design system Master data layer Traceable handoff Scalable development
Approach

ออกแบบสิ่งที่ยังไม่ถูกนิยามให้จับต้องได้เร็วพอสำหรับการตัดสินใจ

Map the real operation

คุยกับ vendor, BA, backend lead และเจ้าหน้าที่ปฏิบัติงาน เพื่อแยกปัญหาหน้าจอออกจากปัญหากระบวนการจริง

Prototype to provoke truth

ทำ mockup แบบเร็วและหยาบพอให้ผู้ใช้ตอบสนองจากของจริง แทนการคุยบนสมมติฐาน

Align UI with data

ทำงานร่วมกับ backend เพื่อจัด UI, data model, master data และข้อจำกัดทางเทคนิคให้ไปทางเดียวกัน

Centralize patterns

จัดระบบ component, variant และ asset management ให้ทีมพัฒนานำไปใช้ซ้ำได้อย่างสม่ำเสมอ

Core modules

โมดูลสำคัญที่เชื่อมงานเอกสาร การวินิจฉัย และการติดตามเคส

G1 Adjudication

โครงสร้างวินิจฉัยเคส

แปลงกระบวนการทำงานจริงเป็นหน้าจอที่ตรวจค่ารักษาและรองรับการตัดสินใจตามแนวทางและกรอบกฎหมาย

G2 Subvention

ลดภาระงานปริมาณสูง

ออกแบบการประมวลผลแบบ asynchronous และ feedback ผ่าน notification เพื่อลดการกดซ้ำและรองรับข้อมูลระดับ 400k+ records

G7 Referral

ติดตามงานหลังส่งต่อ

เพิ่ม tracking layer กลางให้เจ้าหน้าที่ค้นหา กลับมาติดตาม และทำงานต่อจากเคสของตัวเองได้โดยไม่ต้องเริ่มใหม่

Paper to screen

จากข้อมูลดิบสู่ insight งานบริการ

เชื่อม contribution, debt tracking, hospital bill validation และการตรวจความสอดคล้องของ rate ให้ตรวจสอบย้อนกลับได้

การคุยให้สมมติฐาน แต่ UI ทำให้ความจริงปรากฏ

Outcome

ผลลัพธ์คือฐานกลางที่ทำให้ระบบซับซ้อนเดินต่อได้

  • กำหนด workflow ที่ผูกกับเคสและสถานะงาน ลดความคลุมเครือระหว่างบทบาท
  • จัด ownership และ handoff ให้ทีมเห็นว่าขั้นตอนไหนเป็นของใครและส่งต่ออย่างไร
  • ลด duplication ใน master data และช่วยให้ implementation ทำได้จริงขึ้น
  • สร้าง design system กลางสำหรับหลาย vendor เพื่อคุมความสม่ำเสมอของ UI และ behavior
  • รองรับงานตรวจค่ารักษา การติดตามหนี้ และการตรวจจับความผิดปกติของการรักษาในภาพรวม
